# 部署方案 -- 单机部署（Windows / Linux）

> 面向本机启动测试，从零到跑通的最短路径。

---

## 一、环境准备

### 1.1 必装软件

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+ | `python --version` 确认 |
| PostgreSQL | 16+ | 需含 pgvector 扩展 |
| pip | 最新 | 随 Python 安装 |

### 1.2 安装 PostgreSQL + pgvector

**Windows：**

1. 下载 [PostgreSQL 16 安装包](https://www.postostgresql.org/download/windows/)，安装时记住密码
2. 安装 pgvector 扩展（项目启动时自动执行 `CREATE EXTENSION IF NOT EXISTS vector`，但需要 pgvector 的 DLL 已安装）
3. pgvector Windows 安装：用 [PostgreSQL Binaries](https://github.com/pgvector/pgvector#windows) 或下载预编译 DLL 放入 PostgreSQL 的 `lib` 和 `share/extension` 目录

**Linux（Ubuntu/Debian）：**

```bash
# 安装 PostgreSQL 16
sudo apt install postgresql-16 postgresql-contrib

# 安装 pgvector
sudo apt install postgresql-16-pgvector

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 1.3 确认 PostgreSQL 可连接

```bash
# Windows / Linux 通用
psql -U postgres -h localhost
# 输入密码后进入 psql 终端即成功
\q  # 退出
```

## 二、项目部署

### 2.1 获取代码

```bash
git clone https://github.com/shouch111/dingdingbased-opsmate.git
cd dingdingbased-opsmate
```

### 2.2 创建虚拟环境（推荐）

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux
source venv/bin/activate
```

### 2.3 安装依赖

```bash
# 生产部署（精确复现，含全部传递依赖）
pip install -r requirements.lock
```

> 首次安装会下载 PyTorch（sentence-transformers 依赖），约 2GB，请耐心等待。

### 2.4 配置环境变量

在 `data/` 目录下创建 `.env` 文件：

```bash
# Windows
copy data\engineers.example.json data\engineers.json
# 然后手动创建 data\.env

# Linux
cp data/engineers.example.json data/engineers.json
touch data/.env
```

`data/.env` 内容（**按必填项配置，可选项按需调整**）：

```env
# ========== 必填 ==========

# LLM API（DeepSeek 推荐，https://platform.deepseek.com 获取 Key）
open_code_go_api=sk-你的API密钥
model=deepseek-chat
base_url=https://api.deepseek.com

# PostgreSQL（密码改成你安装时设置的）
PG_HOST=localhost
PG_PORT=5432
PG_USER=postgres
PG_PASSWORD=你的PostgreSQL密码
PG_DATABASE=ops_agent

# API 鉴权（三把 Key，用随机字符串，互不相同）
API_KEY_SERVICE=随机字符串1
API_KEY_READONLY=随机字符串2
API_KEY_ADMIN=随机字符串3

# ========== 可选（有默认值，一般不用改）==========

# 模型路由
MODEL_SIMPLE=deepseek-chat
MODEL_MEDIUM=deepseek-chat
MODEL_HARD=deepseek-reasoner

# LLM 超时（秒）
LLM_REQUEST_TIMEOUT=60
LLM_REQUEST_TIMEOUT_HARD=120

# LLM 重试/熔断
LLM_RETRY_MAX_ATTEMPTS=3
LLM_RETRY_MIN_WAIT=1
LLM_RETRY_MAX_WAIT=8
LLM_CIRCUIT_FAILURE_THRESHOLD=5
LLM_CIRCUIT_RECOVERY_SECONDS=60

# 记忆系统
MEMORY_ENABLED=true
MEMORY_SEARCH_TOP_K=3

# 预处理
INTENT_LLM_FALLBACK=true

# 日志
LOG_LEVEL=INFO
LOG_FILE=data/logs/ops_agent.log
LOG_MAX_SIZE=10
LOG_BACKUP_COUNT=5

# 定时提醒
REMINDER_INTERVAL_MINUTES=30
REMINDER_MAX_COUNT=3

# ========== 钉钉（按需配置，不配则跳过钉钉功能）==========
DINGTALK_CLIENT_ID=
DINGTALK_CLIENT_SECRET=
API_MESSAGE_URL=http://localhost:8000/api/v1/message
```

### 2.5 配置工程师名单

编辑 `data/engineers.json`，填入实际工程师信息：

```json
[
  {
    "name": "张三",
    "staff_id": "",
    "skills": ["打印机", "电脑硬件", "Windows系统", "网络"],
    "mobile": "13800000001",
    "dingtalk_user_id": "",
    "available": true
  }
]
```

> 首次启动时自动迁移到数据库，之后改工程师信息直接改数据库。

### 2.6 确认知识库

`data/knowledge/` 下已有 15 个 `.md` 知识库文档。如需新增，直接放入 `.md` 文件即可，系统会自动同步。

## 三、启动

### 3.1 启动命令

```bash
cd data
python -m src.main
```

### 3.2 启动时自动执行

```
1. 连接 PostgreSQL，创建 ops_agent 数据库（如不存在）
2. 安装 pgvector 扩展
3. 建表（engineers/tasks/feedbacks/memories/knowledge_docs）
4. 创建索引（复合索引 + HNSW 向量索引）
5. 迁移 engineers.json -> engineers 表
6. 迁移 difficulty 值（easy -> simple）
7. 加载 Embedding 模型（text2vec-base-chinese，首次约 30 秒）
8. 启动定时提醒调度器
9. 启动 FastAPI（端口 8000）
10. 如配置了钉钉，后台线程启动钉钉 Stream 连接
```

## 四、验证

### 4.1 健康检查（无需鉴权）

```bash
curl http://localhost:8000/health
# 返回：{"status":"ok"}
```

### 4.2 提交工单（需 service Key）

```bash
curl -X POST http://localhost:8000/api/v1/message ^
  -H "X-API-Key: 你的API_KEY_SERVICE值" ^
  -H "Content-Type: application/json" ^
  -d "{\"source\":\"api\",\"sender_name\":\"测试用户\",\"content\":\"打印机连不上\"}"
```

Linux 用 `\` 换行：

```bash
curl -X POST http://localhost:8000/api/v1/message \
  -H "X-API-Key: 你的API_KEY_SERVICE值" \
  -H "Content-Type: application/json" \
  -d '{"source":"api","sender_name":"测试用户","content":"打印机连不上"}'
```

预期返回包含 `response`（解决方案）和 `task_no`（任务编号）。

### 4.3 查看任务列表（需 admin 或 readonly Key）

```bash
curl -H "X-API-Key: 你的API_KEY_ADMIN值" http://localhost:8000/tasks
```

### 4.4 查看日志

```bash
# 控制台直接看，或查看日志文件
type data\logs\ops_agent.log    # Windows
tail -f data/logs/ops_agent.log  # Linux
```

## 五、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError` | 依赖未安装 | `pip install -r requirements.lock` |
| `psycopg2.OperationalError: connection refused` | PostgreSQL 未启动 | 启动 PostgreSQL 服务 |
| `permission denied to create extension "vector"` | pgvector 未安装 | 安装 pgvector 扩展（见 1.2） |
| `LLM_API_KEY 未配置` 警告 | `.env` 中 `open_code_go_api` 为空 | 填入 DeepSeek API Key |
| 启动卡在"加载 text2vec-base-chinese 模型" | 首次下载模型（约 500MB） | 等待下载完成，或设置 `HF_HUB_OFFLINE=1` 用本地缓存 |
| `401 未授权` | 请求未带 `X-API-Key` | 请求头加 `X-API-Key: 对应的key值` |
| `403 权限不足` | Key 角色不匹配 | 写接口用 service/admin key，读接口用 readonly/admin key |
| 钉钉不回复 | 未配置 `DINGTALK_CLIENT_ID` | 按需配置钉钉机器人（可选功能） |

## 六、进程管理（可选）

### Windows -- 用 NSSM 托管

```bash
# 下载 nssm.exe 后
nssm install ops-agent
# Application Path: C:\path\to\venv\Scripts\python.exe
# Arguments: -m src.main
# Startup directory: C:\path\to\data
nssm start ops-agent
```

### Linux -- 用 systemd 托管

创建 `/etc/systemd/system/ops-agent.service`：

```ini
[Unit]
Description=Ops Agent
After=postgresql.service

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/dingdingbased-opsmate/data
ExecStart=/path/to/venv/bin/python -m src.main
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl start ops-agent
sudo systemctl enable ops-agent
sudo systemctl status ops-agent
```
